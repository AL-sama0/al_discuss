# 常见问题 (FAQ)

## 关于论坛

### Q：F.U.C.K. Forum 是什么？
A：F.U.C.K. Forum 是一个基于 GitHub Discussions 的开源社区，致力于帮助人们从**专注(Focus)、理解(Understand)、创造(Create)、坚持(Keep)**四个维度实现成长。不受限制的话题范围，但通过不同的分区引导讨论朝向有价值的方向。

### Q：这个论坛和普通论坛有什么区别？
A：三个主要区别：
1. **完全免费开源** - 无需付费，代码公开，托管在 GitHub
2. **围绕成长设计** - 分区不是按技术分类，而是围绕成长的完整闭环
3. **不限话题** - 虽然有分区建议，但你自由讨论任何话题

### Q：我需要注册吗？
A：不需要额外注册。使用你已有的 GitHub 账号直接登录即可参与讨论。如果你没有 GitHub 账号，需要先去 [github.com](https://github.com) 注册一个。

### Q：论坛有审核制度吗？
A：有基本的社区准则（见使用指南）。任何违反准则的内容（如垃圾信息、人身攻击、硬广告等）都可能被删除。但我们的审核理念是包容和建设性的。

## 关于分区

### Q：四个分区分别适合讨论什么？
A：简单对比：

| 分区 | 适合话题 | 典型帖子 |
|------|---------|---------|
| 🎯 Focus | 时间/精力/目标 | "寻打卡搭子"、"推荐时管工具" |
| 💡 Understand | 学习/思考/分析 | "读书笔记"、"拆解商业模式" |
| 🚀 Create | 作品/项目 | "分享我的 App"、"设计作品展示" |
| 💪 Keep | 打卡/复盘 | "100天跑步打卡"、"季度复盘" |

### Q：我的话题属于多个分区，应该发在哪里？
A：选择最主要的一个分区发布。例如"从失败中学到的 Python 设计模式"可能既适合 Understand，也适合 Create，但如果重点是分享作品，就发在 Create；如果重点是知识分享，就发在 Understand。

### Q：是否可以在多个分区发布同一个讨论？
A：不建议。但如果内容确实相关多个分区，可以在主要分区发布，然后在其他分区的一个讨论中留个链接和简短描述。

### Q：分区之间的讨论会混乱吗？
A：不会。GitHub Discussions 有明确的分区划分，访问者可以按分区浏览，不会看到混乱。

## 关于讨论

### Q：一个讨论要多观点讨论才能关闭吗？
A：没有硬规定。发起者可以随时关闭讨论，或者讨论可以长期进行。我们鼓励讨论在得出共识或发起者满足后自然结束。

### Q：我发布的内容可以删除吗？
A：可以。你对自己发布的内容有完全的控制权，可以随时编辑或删除。讨论本身可以转换为"已解决"或其他状态。

### Q：讨论会保存多久？
A：永久保存。只要 GitHub 仓库存在，讨论就会一直存在。这也是为什么高质量的讨论特别有价值。

### Q：我能匿名讨论吗？
A：不能。GitHub Discussions 需要登录，所有讨论都与你的 GitHub 账号关联。但你可以创建一个昵称账号来实现某种程度的匿名。

## 关于参与

### Q：我是初学者，能参与吗？
A：完全可以！实际上，初学者的问题和经历往往很有价值。提出问题、分享学习过程，都是受欢迎的。

### Q：我没有"高大上"的项目/想法，可以分享吗？
A：完全可以。小项目、小想法、小进步都有价值。Create 区的一个核心理念就是"Build in Public"——过程本身就很有价值，不必等到完美再分享。

### Q：如何让我的讨论获得更多关注？
A：几个建议：
1. **标题清晰具体** - 不是"求助"，而是"如何用 Notion 管理个人知识库"
2. **提供价值** - 分享具体经验或问题，而不是笼统的感叹
3. **参与他人讨论** - 你的参与会让更多人看到你
4. **定期更新** - 特别是打卡贴，定期更新会保持热度
5. **适时总结** - 讨论进行一段时间后，用总结帖汇总关键点

### Q：别人的回答不准确，我能纠正吗？
A：可以。礼貌地指出并解释为什么你认为不准确。讨论的目标是找到最佳答案，而不是谁对谁错。

### Q：什么时候应该新开一个讨论而不是回复？
A：一般规则：
- 新问题 → 新讨论
- 同一问题的不同方面 → 同一讨论的新回复
- 基于已有讨论的衍生话题 → 新讨论，但可以在旧讨论中链接

## 技术相关

### Q：支持哪些 Markdown 语法？
A：GitHub Flavored Markdown (GFM) 的所有功能：代码块、表格、嵌入、数学公式、任务列表等。详见使用指南。

### Q：可以发布图片和视频吗？
A：可以。在讨论中可以拖拽上传图片，也可以链接外部视频（YouTube、Vimeo 等）。

### Q：讨论支持实时通知吗？
A：支持。你可以在 GitHub 设置中配置通知方式（邮件、浏览器通知等）。

### Q：讨论可以搜索吗？
A：可以。GitHub Discussions 支持全文搜索，也可以按分区、关键词筛选。

## 关于社区

### Q：谁能成为社区管理员？
A：目前由论坛创建者维护。如果论坛成长到一定规模，会考虑邀请活跃成员成为版主。

### Q：论坛可能关闭吗？
A：虽然不排除各种可能，但既然代码完全开源，即使官方停运，任何人也能 fork 并继续维护。

### Q：论坛有商业化计划吗？
A：暂无。论坛的目的是提供一个免费开放的社区。会持续免费和开源。

### Q：可以在论坛上打广告吗？
A：硬广告（如"立即购买 XXX"）不欢迎。但与讨论相关的项目分享、工具推荐等是可以的。关键是提供价值，而不仅仅是推销。

## 其他

### Q：论坛的名字"F.U.C.K."是什么意思？
A：是 **F**ocus, **U**nderstand, **C**reate, **K**eep 的首字母缩写。这四个动词代表从梦想走向现实的完整闭环。虽然字母组合看起来"不庄重"，但这正是论坛的个性——用不同的方式传达变革的力量。

### Q：我有建议或想报告 Bug，应该怎么做？
A：在 [GitHub Issues](https://github.com/AL-sama0/al_discuss/issues) 中提交，或者在 Keep 区发起讨论。我们欢迎任何建设性的反馈。

### Q：我能贡献代码吗？
A：完全可以！这是一个开源项目。欢迎 Fork、改进和提交 Pull Request。详见 GitHub 仓库的 CONTRIBUTING 指南。

### Q：论坛支持其他语言吗？
A：目前以中文为主。但如果有需求，我们可以考虑多语言支持。在 GitHub Issues 中提出想法。
