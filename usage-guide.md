# 使用指南

## 如何在 F.U.C.K. Forum 上讨论

### 第一步：访问 GitHub Discussions

所有的讨论都基于 GitHub Discussions 进行。你有两个选择：

1. **直接点击各分区页面下方的评论区**（推荐）
2. **访问仓库的 Discussions 标签**：https://github.com/AL-sama0/al_discuss/discussions

### 第二步：选择或创建分区

F.U.C.K. Forum 有四个主要分区，每个分区都有特定的用途：

| 分区 | 用途 | 适合讨论 |
|------|------|---------|
| 🎯 Focus | 目标与精力管理 | 时间管理、打卡搭子、戒除成瘾 |
| 💡 Understand | 知识输入与深度思考 | 教程、读书笔记、行业分析 |
| 🚀 Create | 实战与作品展示 | 项目、代码、设计、文章、产品 |
| 💪 Keep | 打卡与复盘反思 | 长期打卡、失败日志、复盘总结 |

### 第三步：发起讨论

1. 在 GitHub Discussions 中点击 "New Discussion"
2. 选择合适的分区
3. 给讨论起一个清晰的标题（例如："寻找番茄钟搭子 - 朝九晚五"）
4. 在描述中提供必要的背景和细节
5. 点击 "Start Discussion"

### 第四步：参与讨论

- 使用 **Markdown** 语法增强你的内容（支持代码块、列表、链接等）
- 添加 **表情反应**来快速表达态度
- 引用他人的回答进行深度讨论
- 分享相关的链接和资源

## 讨论礼仪

✅ **请这样做：**
- 使用清晰、具体的标题
- 提供足够的背景信息
- 引用其他讨论时提供链接
- 用表情回复表达简短的想法
- 感谢有帮助的回答

❌ **请不要这样做：**
- 发布重复的讨论
- 用纯表情或"赞"作为回复
- 在错误的分区发布
- 发布商业广告（非讨论相关）
- 给出有攻击性的评论

## 获得最好的回答

### 提出好问题的技巧

1. **使用清晰的标题** - 一个好的标题能帮助人理解问题核心
   - ❌ "求助" 
   - ✅ "React hooks 中如何在 useEffect 中正确处理异步操作？"

2. **提供背景信息** - 为什么你想知道这个？你已经尝试了什么？
   
3. **包含代码示例** - 如果与代码有关，提供最小化可重现的例子

4. **明确你想要的帮助** - 是想要最佳实践？是想解决 Bug？

5. **展示你的努力** - 说明你已经搜索过或尝试过什么

### 回答他人问题的建议

- 首先理解问题再回答
- 提供具体例子而非笼统的建议
- 解释"为什么"而不仅仅是"怎么做"
- 提供多个方案让提问者选择

## 常见问题

**Q：我能在多个分区发布同一个讨论吗？**  
A：不建议。找最合适的一个分区。如果真的相关多个分区，可以在一个地方发，在其他地方留链接。

**Q：讨论多久没人回复需要跟进？**  
A：通常热门讨论会在 24 小时内有回复。如果没有，可以：
- 确保标题清晰
- 补充更多细节
- 尝试在分区的另一个讨论中提及

**Q：如何删除我的讨论？**  
A：在讨论详情页的"..."菜单中可以选择删除。每个讨论也可以转换为讨论 (Discussion) 或普通讨论。

**Q：能用中文吗？**  
A：完全可以！论坛支持中文。正体、繁体都都没问题。
